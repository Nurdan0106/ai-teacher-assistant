# ai-teacher-assistant
Проект: «Сократ» — AI-репетитор по школьной информатике
Контекст: Прототип для магистерской/кандидатской диссертации на тему
«Исследование влияния инновационных технологий на преподавание школьной информатики»

🎯 Цель проекта
Создать полноценное веб-приложение, где школьники (5–11 класс) могут получить помощь по информатике от AI-ассистента, который работает по методу Сократа — не даёт готовых решений, а ведёт к ответу через наводящие вопросы.

🧠 Главный педагогический принцип
AI НИКОГДА не должен:

Писать готовый код решения задачи
Давать полный ответ сразу
Решать задачу за ученика

AI ВСЕГДА должен:

Разбивать задачу на шаги и спрашивать, с какого начать
Задавать наводящие вопросы при затруднении
Проверять ученика вопросом: «А что будет, если...?»
Хвалить за ход мысли, а не только за правильный ответ
Адаптировать сложность под класс ученика


🏗️ Архитектура
Frontend

Стек: HTML + CSS + Vanilla JavaScript (без фреймворков для простоты)
Стиль: Editorial / Academic — тёплая бумажная палитра, шрифт Fraunces
Адаптивность: mobile + desktop

Backend (ОБЯЗАТЕЛЬНО)

Стек: Node.js + Express
Назначение: проксировать запросы к Claude API, скрывать API-ключ
Endpoint: POST /api/chat принимает { grade, messages } и возвращает ответ

AI

Модель: Claude Sonnet 4 (claude-sonnet-4-20250514)
API: Anthropic Messages API
Контроль: через системный промпт (см. ниже)

🤖 Педагогический промпт (ядро проекта)
Промпт должен содержать:

Роль: «Ты — Сократ, AI-репетитор»
Возрастной контекст (7 вариантов для 5–11 классов):

5 кл — примеры из игр и мультиков
7 кл — базовые термины с объяснением
9 кл — подготовка к ОГЭ
11 кл — подготовка к ЕГЭ и олимпиадам


Список запретов (готовый код, прямые ответы)
Список правил (наводящие вопросы, разбиение на шаги)
Примеры диалогов (плохо vs хорошо):

Ученик: «Реши задачу: найти максимум в массиве»
❌ Плохо: «Вот код: max_val = max(arr)»
✅ Хорошо: «Представь, что у тебя 5 карточек с числами.
Как ты сам нашёл бы самую большую? Опиши словами.»

Формат ответов:

Коротко (3–5 предложений)
Дружелюбный тон, на «ты»
Заканчивать вопросом
Поддержка <em> и inline-кода в backticks




🔧 Функциональные требования
Frontend

 Выбор класса сохраняется в localStorage
 История чата хранится в памяти сессии
 Autoresize для textarea
 Enter = отправка, Shift+Enter = новая строка
 Индикатор печати (три точки) во время ожидания ответа
 Обработка ошибок сети с понятным сообщением
 4 кнопки-подсказки с типичными вопросами
 Анимации появления сообщений

Backend

 Express сервер на порту 3000
 CORS настроен для localhost
 POST /api/chat с валидацией входа
 Rate limiting (10 запросов/минуту с IP)
 Логирование запросов (без содержимого, только метаданные)
 .env файл для ANTHROPIC_API_KEY
 Graceful error handling

DevOps

 package.json с scripts: dev, start
 .gitignore исключает .env и node_modules
 README с инструкцией по запуску
 .env.example как шаблон
